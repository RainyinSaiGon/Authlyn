# Security Policy

Authlyn is an identity and access management project, so security reports are important.

## How to report a vulnerability

Please report security issues privately instead of opening a public issue.

Preferred report contents:
- a short summary of the issue
- affected endpoint, feature, or module
- impact summary
- steps to reproduce, if safe to share
- your contact details for follow-up

## Do not include in public issues

Do not post any of the following in a public GitHub issue or pull request:
- passwords or secrets
- API keys, tokens, or refresh tokens
- database dumps
- live user data
- exploit proof-of-concept details that could harm real systems

## Response expectations

A maintainer should review the report as soon as practical and coordinate next steps privately.

If the issue is not security-sensitive, please use the normal issue templates in `.github/ISSUE_TEMPLATE/`.
